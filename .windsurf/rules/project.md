# Workspace Rules — Small Wins Automation

## GitHub Safety Reminder (Always)
- After meaningful code changes, always remind me to commit and push to GitHub.
- Before ending a coding task, explicitly confirm push status:
  - "Committed and pushed"
  - or "Not pushed yet" with exact next command to run.
- Prefer one clear commit per completed task so changes are easy to trace.

## Multi-Folder Protection
- I have multiple small-wins-automation folders with valuable work.
- Always include the exact repo path when giving git commands.
- If push cannot be completed, provide copy-paste commands for commit + push.
